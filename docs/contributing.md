# Contributing

We welcome contributions! Here's how to suggest a project.

## Submitting a Project

1. **Open an issue** with the title: `[Project Name] - Brief description`
2. Include:
   - Project URL (GitHub repo preferred)
   - Why it's emerging (what signal caught your attention?)
   - Any metrics: HN points, GitHub stars, community discussion links
   - Which category it fits best

3. Or **submit a PR** directly — edit the appropriate category markdown file and add your entry following the format below.

## Entry Format

```markdown
### [Project Name](https://github.com/org/repo)
> One-line description of what it does.

- **Discovered**: YYYY-MM-DD
- **Signal**: 🔥/⚡/🌱 [metrics]
- **Tags**: `tag1` `tag2`

---
```

## Signal Thresholds

| Emoji | When to use |
|-------|-------------|
| 🔥 Hot | >500 HN points or >10k GitHub stars |
| ⚡ Rising | >100 HN points or >1k GitHub stars |
| 🌱 Early | Everything else with genuine promise |

## Quality Guidelines

- **Do** submit projects you've actually looked at or used
- **Do** include the community signal that caught your attention
- **Do** check that the project isn't already listed
- **Don't** submit your own project without genuine community traction
- **Don't** submit abandoned/archived repos
- **Don't** submit closed-source / paid-only tools

## Categories

| Category | Scope |
|----------|-------|
| AI Agents & Frameworks | Agent frameworks, MCP tools, LLM infrastructure |
| CLI Tools | Terminal utilities, command-line tools |
| Developer Tools | IDEs, build tools, package managers, dev workflow |
| Open Source Infrastructure | Networking, databases, platforms, system tools |

If a project doesn't fit any category, suggest a new one in your issue/PR.
