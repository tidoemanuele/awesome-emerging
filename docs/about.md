# About Awesome Emerging

## Mission

Track and curate **emerging open source projects that actually matter** — the ones gaining real traction in developer communities, not just the ones with the best marketing.

## Why This Exists

There are thousands of awesome lists on GitHub. Most are static catalogs: someone adds a project, it sits there forever, and there's no signal about whether it matters or not.

Awesome Emerging is different:

- **Momentum-tracked** — every entry includes the community signal at time of discovery (HN points, GitHub stars, discussion volume)
- **Time-stamped** — you can see *when* a project started gaining attention, not just that it exists
- **Signal-filtered** — projects earn their spot through demonstrated community interest, not self-promotion
- **Actively curated** — sourced from daily research across HackerNews, GitHub trending, and developer communities

## Methodology

Projects are discovered through daily research that monitors:

1. **HackerNews** — trending posts, Show HN launches, and high-signal discussions
2. **GitHub Trending** — star velocity across languages and timeframes
3. **Developer communities** — Reddit, Discord, Twitter/X, and specialized forums

### Inclusion Criteria

- Must be open source or have a significant open-source component
- Must show genuine community traction (not just stars — engagement, forks, discussion)
- Must solve a real problem or introduce a novel approach
- Preference for projects in active development

### Exclusion Criteria

- Abandoned or archived projects
- Pure marketing / landing pages with no substance
- Projects that are already well-established and widely known (React, Kubernetes, etc.)
- Paid-only tools with no open-source component

## Signal Rating System

| Rating | Criteria |
|--------|----------|
| 🔥 **Hot** | >500 HN points or >10k GitHub stars — massive community attention |
| ⚡ **Rising** | >100 HN points or >1k GitHub stars — clear upward trajectory |
| 🌱 **Early** | Promising project with growing interest, still early |
